---
title: "gsoc2 secrets"
description: "Perform CRUD operations with Gsoc2 secrets"
---

```
gsoc2 secrets
```

## Description
This command enables you to perform CRUD (create, read, update, delete) operations on secrets within your Gsoc2 project. With it, you can view, create, update, and delete secrets in your environment.

### Sub-commands 
<Accordion title="gsoc2 secrets" defaultOpen="true">
  Use this command to print out all of the secrets in your project

  ```bash
  $ gsoc2 secrets
  ```

  ### Environment variables
  <Accordion title="GSOC2_TOKEN">
    Used to fetch secrets via a [service token](/documentation/platform/token) apposed to logged in credentials. Simply, export this variable in the terminal before running this command.

    ```bash
    # Example 
    export GSOC2_TOKEN=st.63e03c4a97cb4a747186c71e.ed5b46a34c078a8f94e8228f4ab0ff97.4f7f38034811995997d72badf44b42ec
    ```
  </Accordion>

  <Accordion title="GSOC2_DISABLE_UPDATE_CHECK">
    Used to disable the check for new CLI versions. This can improve the time it takes to run this command. Recommended for production environments.
    
    To use, simply export this variable in the terminal before running this command.

    ```bash
    # Example 
    export GSOC2_DISABLE_UPDATE_CHECK=true
    ```
  </Accordion>

  ### Flags 
  <Accordion title="--expand">
    Parse shell parameter expansions in your secrets

    Default value: `true`
  </Accordion>

  <Accordion title="--env">
    Used to select the environment name on which actions should be taken on

    Default value: `dev`
  </Accordion>
  <Accordion title="--path">
    The `--path` flag indicates which project folder secrets will be injected from.

    ```bash
    # Example
    gsoc2 secrets --path="/" --env=dev
    ```
  </Accordion>

</Accordion>


  </Accordion>

<Accordion title="gsoc2 secrets set">
This command allows you to set or update secrets in your environment. If the secret key provided already exists, its value will be updated with the new value. 
If the secret key does not exist, a new secret will be created using both the key and value provided.

```bash
$ gsoc2 secrets set <key1=value1> <key2=value2>...

## Example 
$ gsoc2 secrets set STRIPE_API_KEY=sjdgwkeudyjwe DOMAIN=example.com HASH=jebhfbwe
```

  ### Flags 
  <Accordion title="--env">
    Used to select the environment name on which actions should be taken on

    Default value: `dev`
  </Accordion>
  <Accordion title="--path">
    Used to select the project folder in which the secrets will be set. This is useful when creating new secrets under a particular path.

    ```bash
    # Example
    gsoc2 secrets set DOMAIN=example.com --path="common/backend"
    ```
  </Accordion>
</Accordion>

<Accordion title="gsoc2 secrets delete">
  This command allows you to delete secrets by their name(s).

  ```bash
  $ gsoc2 secrets delete <keyName1> <keyName2>...

  ## Example 
  $ gsoc2 secrets delete STRIPE_API_KEY DOMAIN HASH
  ```

  ### Flags 
  <Accordion title="--env">
    Used to select the environment name on which actions should be taken on

    Default value: `dev`
  </Accordion>
  <Accordion title="--path">
    The `--path` flag indicates which project folder secrets will be injected from. 

    ```bash
    # Example
    gsoc2 secrets delete <keyName1> <keyName2>... --path="/"
    ```
  </Accordion>
</Accordion>

<Accordion title="gsoc2 secrets folders">
  This command allows you to fetch, create and delete folders from within a path from a given project.

  ```bash
  $ gsoc2 secrets folders
  ```

  ### sub commands 
  <Accordion title="get">
    Used to fetch all folders within a path in a given project
    ```
    gsoc2 secrets folders get --path=/some/path/to/folder 
    ```
    #### Flags 
    <Accordion title="--path">
      The path from where folders should be fetched from

      Default value: `/`
    </Accordion>

    <Accordion title="--token">
      Fetch folders using the Gsoc2 service token

      Default value: ``
    </Accordion>

  </Accordion>

  <Accordion title="create">
    Used to create a folder by name within a path. 
    ```
    gsoc2 secrets folders create --path=/some/path/to/folder --name=folder-name
    ```
    ### Flags 
    <Accordion title="--path">
      Path to where the folder should be created

      Default value: `/`
    </Accordion>

    <Accordion title="--name">
      Name of the folder to be created in selected `--path`

      Default value: ``
    </Accordion>
  </Accordion>

  <Accordion title="delete">
    Used to delete a folder by name within a path. 
    ```
    gsoc2 secrets folders delete --path=/some/path/to/folder --name=folder-name
    ```
    ### Flags 
    <Accordion title="--path">
      Path to where the folder should be created

      Default value: `/`
    </Accordion>

    <Accordion title="--name">
      Name of the folder to be deleted within selected `--path` 

      Default value: ``
    </Accordion>
  </Accordion>  
</Accordion>

<Accordion title="gsoc2 secrets generate-example-env">
This command allows you to generate an example .env file from your secrets and with their associated comments and tags. This is useful when you would like to let 
 others who work on the project but do not use Gsoc2 become aware of the required environment variables and their intended values.

To place default values in your example .env file, you can simply include the syntax `DEFAULT:<value>` within your secret's comment in Gsoc2. This will result in the specified value being extracted and utilized as the default.

```bash
$ gsoc2 secrets generate-example-env

## Example 
$ gsoc2 secrets generate-example-env > .example-env
```

  ### Flags 
  <Accordion title="--env">
    Used to select the environment name on which actions should be taken on

    Default value: `dev`
  </Accordion>
</Accordion>
